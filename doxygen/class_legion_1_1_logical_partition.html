Detailed Description

Logical partition objects defines handles to the actual logical partitions maintained by the runtime. Logical partitions are defined by a triple consisting of the index partition, field space, and region tree ID of the logical partition. These three values are sufficient to name every logical partition created in a Legion program.

Logical partition objects can be copied by values and stored in data structures. Only the Legion runtime is able to create non-empty logical partitions.
